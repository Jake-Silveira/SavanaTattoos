# SavanaTattoos

A tattoo studio website with public inquiry form and admin dashboard, built on Vercel + Supabase.

## Features

- **Public Inquiry Form** — Tattoo intake form with body placement, size category (XS–XXL), color preferences, cover-up/original selection
- **Portfolio Gallery** — Display tattoo work with carousel and grid layouts
- **Flash Gallery** — Showcase flash tattoo designs
- **Client Reviews** — Auto-rotating testimonial carousel
- **Admin Dashboard** — Full management system for leads, scheduling, client profiles, and site content
- **Lead Management** — Track inquiries from pending through confirmed
- **Calendar Scheduler** — Tattoo-appropriate time slots based on size categories
- **Email Notifications** — Confirmation emails via Resend

## Core Technologies

- **Frontend:** Vanilla JavaScript (ES6+), HTML5, CSS3 (CSS custom properties for theming)
- **Backend/Database:** [Supabase](https://supabase.com/) for PostgreSQL storage, Authentication, and S3-compatible Storage
- **Serverless Functions:** [Vercel Functions](https://vercel.com/docs/functions) for post-submission logic and administrative data management
- **Hosting:** [Vercel](https://vercel.com/) (Free Tier)

## Commands

```bash
npm run build    # runs node build.js — copies assets to public/, injects env vars into HTML
npm start        # npx serve public — serve the built output locally
```

No linter, formatter, typechecker, or test runner exists.

## Setup & Deployment

### Prerequisites

- Node.js (for the build script and local development)
- Supabase Project (URL, Anon Key, and Service Role Key)
- Optional: Resend account (email delivery), Discord webhook (notifications)

### Local Development

1. **Environment Setup:**
   ```bash
   cp .env.example .env
   # Edit .env with your SUPABASE_URL, SUPABASE_ANON_KEY, SUPABASE_SERVICE_ROLE_KEY, and other keys
   ```

2. **Build the Project:**
   Cleans `public/`, copies assets, and injects variables.
   ```bash
   npm run build
   ```

3. **Run Locally:**
   ```bash
   npm start
   ```

### Database Setup

See [`SETUP.md`](./SETUP.md) for complete Supabase provisioning:
- Create Supabase project
- Run schema SQL (creates all tables, RLS policies)
- Create storage buckets (`portfolio`, `flash-images`, `client-photos`)
- Create admin user with `role = 'admin'` in `admin_profiles`

### Deployment — Vercel

Deploy to Vercel with the following configuration (also defined in `vercel.json`):

- **Build Command:** `node build.js`
- **Output Directory:** `public/`
- **Routing:** `/admin` rewrites to `admin.html`
- **Functions:** All routes in `api/` are deployed as serverless functions
- **Max Duration:** 10s (Vercel Hobby plan limit)

Configure Vercel project environment variables:
- `SUPABASE_URL`, `SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY`
- `RESEND_API_KEY` (optional), `FROM_EMAIL`, `ADMIN_EMAIL`
- `DISCORD_WEBHOOK_URL` (optional)

## Architecture

- **Static site** served from `public/` (generated by `build.js`). Source files are at root level (`index.html`, `script.js`, `admin.js`, `styles.css`, `utils.js`).
- **Vercel serverless functions** in `api/` — see `vercel.json` for route mapping.
- **Database:** Supabase PostgreSQL with RLS. No backend ORM — direct queries from browser JS and API routes.
- **Env var injection:** `build.js` reads `.env` (local) or `process.env` (Vercel), replaces `<!-- ENV_PLACEHOLDER_START -->` / `<!-- ENV_PLACEHOLDER_END -->` blocks in `index.html` and `admin.html` with `window.__ENV__`.

### Public Site

The public site (`index.html`) serves as a tattoo studio brochure with:
- Hero section with studio branding
- Portfolio gallery carousel
- Flash tattoo design grid
- Client reviews carousel
- Services overview
- Contact information
- Inquiry booking form (generates leads)

### Admin Dashboard

The admin dashboard (`admin.html`) provides tattoo studio management:
- **Appointments Panel** — View leads, update status, create bookings
- **Scheduler Panel** — Calendar view with time blocking, manage availability
- **Client Profiles Panel** — Manage client records, notes, tattoo history
- **Site Management Panel** — Manage portfolio, flash gallery, reviews, site feedback

### Client-Side Data Access

The frontend communicates directly with Supabase using the browser SDK for read operations and administrative updates.

### Security (RLS)

Security is enforced at the database level using Supabase **Row Level Security (RLS)**:

- **Public (Anonymous):** Can `INSERT` new booking inquiries, and `SELECT` active reviews and gallery items.
- **Authenticated (Admin):** Can `SELECT`, `INSERT`, `UPDATE`, and `DELETE` entries across all tables.
- **Admin API routes** use the Supabase Service Role key to bypass RLS.

### Mobile-First Design

The site and dashboard are fully optimized for mobile devices, including a responsive calendar and time-grid scheduler.

### Dark Tattoo Theme

The design uses a dark/edgy aesthetic with gold accent (`#c9a87c`) on dark backgrounds, consistent with tattoo studio branding.

## Development Conventions

- **RLS-first security:** All Supabase access uses RLS policies. `anon` can INSERT public forms; `authenticated` gets full access. Service role key bypasses RLS for serverless functions.
- **Build step is required:** `public/` must be rebuilt whenever `.env` changes or new source files are added. Update `build.js` arrays accordingly.
- **`public/` is both tracked and rebuilt:** `build.js` deletes and recreates `public/` on each build, but the directory is committed to git. This means committed `public/` is a fallback for static assets.

## Security Practices

1. **RLS Enforcement:** All database access is governed by strict Row Level Security policies.
2. **Service Role Key:** Sensitive operations are handled via Vercel functions using the Supabase Service Role key. Admin API routes use `verifyAdmin()` to check JWT + admin role.
3. **Hardened Login:** The admin login flow uses persistent session state and protects against accidental submissions.
